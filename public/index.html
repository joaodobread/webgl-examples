  • O que é o WebGL:
WebGL é basicamente uma API de rasterização e não uma API 3D.

O WebGL é chamado de API de rasterização basicamente pela quantidade de matemática 3D que você precisa saber para que utilize-a;

Basicamente a api WebGL não cria gráficos em 3D e não se encaixa como uma biblioteca 3D. Diferentemente de outras bibliotecas como ThreeJs o WebGL você precisa entender de matrizes, coordenadas normalizadas, frustums, produtos cruzados, produtos escalares, interpolação variável e cálculos especulares de iluminação e todo os tipos de outras coisas que geralmente levam anos para serem adquiridos.

Em relação ao canvas o WebGL não tem grandes diferenças, principalmente a nível de codificação. O código dos dois não tem tamanhos tão diferentes, mas a diferença mais clara é que o canvas faz um loop sobre os vértices fornecidos, já o WebGL faz o mesmo, só que a matemática fornecida está em GLSL e é diretamente executada pela GPU;
